浅谈我的hexo历程

发表于 2017-10-19


　　emmm…大概是上周五晚上,我看到了学霸同学的制作精美的博客，瞬间萌生了自己也搞一个的念头。于是，从上周六开始，我便开始了我的“跌跌撞撞”hexo历程。也是我天性愚笨，皮了快一周才将博客大体完善，感慨良多，想写个博客纪念一下，也算是我的处女文吧~~

　　我主要搭建博客都是参考了这篇博客，他上面讲的还是比较全面的。而我碰到不会的地方的时候，翻翻评论区也会有惊喜，往往问题就解决了。搭建博客主要就是利用hexo与github page的关联，这样就可以将本地的文章等发布到网络上，与大家分享。大家通过访问yourname.github.io网址，就可以看到你的微博啦！当然，我们也可以自己注册一个域名，实现通过自己个性域名进行访问，具体可见这篇博客。

　　关联好以后就是对于hexo主题的选取，这就仁者见仁、智者见智了。我比较喜欢next风格，一是装饰博客的时候比较自主，二是看起来清爽简洁。之后，我们可以对于hexo博客的一些小细节和更多特效进行捣鼓，更多特效可以参考这篇博客，他上面介绍的特效可谓非常丰富了，能让大家捣鼓不少时间呢！而这也是博客最好玩的地方，当你看到许多特效在你手中实现时，成就感无疑是满满的。

　　当以上都完成的差不多时，博客也就基本搭建好了，于是我就开始学着写博客。博客的编写主要是运用Markdown语法，关于Markdown语法的学习，我推荐学霸同学的博客。他上面列出了很多常用语法，对我学习写博客很有帮助。

现在总结起来，感觉搭建博客挺简单的，不过当时我着实遇到不少坑：
　　1、没能注意到配置文件（就是.yml文件）中的格式问题，该打空格的地方没打空格，于是输入部署指令的时候，控制台毫无反应，hexo和github page也始终关联不上。我找了半天错误原因还找不到，当时可把我给愁死了，浪费了我不少时间。
　　2、将自己的域名和github page关联的时候，需要在\source下建立一个命名为“CNAME”的无扩展名文件，以保证每次部署以后github能保存自己的域名。可我查阅了一篇有误的文档，上面教我命名为“GNAME”,于是emmm…男默女泪，知道真相的我想报警。
　　3、在部署的时候，系统可能会报错，比如“ERROR Script load failed: themes/hexo-theme-next/scripts/tags/exturl.js ”，虽然这类错误并不影响程序运行，但是对于我们这些有强迫症的程序猿来说，就非常难受。这时候你只需要输入npm install hexo-util –save指令，错误就会消失了。
　　4、这个也不算坑吧，是一个提醒。就是在设置ssh密钥的时候，不知道为啥，不能使用控制台，只能使用git bash去完成。
　　5、在设置头像动画的过程中，emmm…我遇到了一个现在还没有解决的坑。我使用和我一个朋友相同的方法，就是在\themes\next\source\css_common\components\sidebar\sidebar-author.styl中加入如下代码： 

.site-author-image:hover {
  -webkit-animation: shake 1s;
  animation: flip 1s;
};

结果，他的头像有动画，我的却没有…弄了一晚上都不知道啥原因，希望知道原因的大神能教教我哈~~
　　先列出这么多坑，想起来了我再补上…（emmm…不过我觉得我应该想不起来了）

　　搭建博客的历程大体就是这样了，真的是痛并快乐着。纠结找bug的时候难受的要死，但是成功了又会欣喜若狂、成就满满！鼓励大家一起来玩哈~~

　　最后我还要感谢上述博客带给我的帮助，没有他们或许我还要再花很久的时间才能搞好，真的非常感谢~~
