Oct_26.取数游戏

发表于 2017-10-29 | 分类于 要写题啊 | 热度: ℃
字数统计: 998 | 阅读时长 ≈ 4

　　欢迎来到“要写题啊”频道~~

　　今天我想分享给大家的题目是——取数游戏！

　　想写本题请戳题目传送门

　　emmm…不要问我为啥26号的题现在才发。一个是最近上党课啥的，事情比较多；二是这道题对于我来说是真的有难度；三是…好吧，其实就是我很懒。-。-！

　　闲话说过了，该来分析题了。这道题乍一看多少有些博弈论的意思，实际我们可以用动态规划（DP）去进行求解。当我开始接触这条题目的时候，没有多少算法基础的我，只能很呆地去思考。我想的是从外部的四个数（开头两个数A、B，末尾两个数C、D）去处理，满足不等式（B - A <= C - D && D - A <= C - D）即应该取A不取D。可经过检验后，发现这样的解题方法并不行，有很多方面思考得不到位。于是，我便开始问度妈，得到了一种解题方法——动态规划（DP）。但是写那篇博客的博主写的很模糊，emmm…大概是他觉得这个东西应该人人都能理解吧。我没办法只能对着他的C++代码去理解动态规划的思想，着实是很难懂，反正我是看了半天，看的极其烦躁、头皮发麻，不过好在后来弄明白了。

　　下面我先把后来自己写的Java代码晒出来，供大家参考

　　(我自认为备注还是非常清晰的哈~)

　　这题主要运用了公式best[i][i + j] = max(sum[i][i + j] - best[i][i + j - 1], sum[i][i + j] - best[i + 1][i + j])去处理（其中best[i][j]表示从第i个数到第j个数先手取的最优解，sum[i][j]表示从第i个数到第j个数的累和，max是一个求两者最大数的方法）。从任意两个相邻的数（如第一个数和第二个数）开始，分别计算了所有不同组（任意两个相邻的数为一组）先手第一次取头和取尾的结果并比较后选取最优解。由两个相邻的数扩大到三个相邻的数再到所有数，每次都能保证是最优解，并且后面的处理过程中会借用前面已经得出的结论，这有点类似于递归。最终就能得到所有数中先手取的最优解，后手的结果便是总和减去先手取的最优解。

　　以上就是今天我给大家带来的取数游戏题目啦，如果大家有什么好的建议或者意见，或是对这道题还有什么疑惑的地方，欢迎与我取得联系~~
